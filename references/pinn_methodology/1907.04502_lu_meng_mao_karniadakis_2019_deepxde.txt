ABSTRACT
--------
Overview of physics-informed neural networks (PINNs): embed a PDE into the NN
loss using automatic differentiation. Algorithm is simple; applies to
integro-differential, fractional, and stochastic PDEs. Inverse problems
solved as easily as forward problems. Proposes a NEW residual-based adaptive
refinement (RAR) method to improve training efficiency. Compares PINN to
FEM. Presents the Python library DeepXDE (forward + inverse, complex
geometry via constructive solid geometry; user code stays compact and
mirrors the math). Demonstrated on five examples.

KEYWORDS: education software · DeepXDE · differential equations · deep
learning · PINNs · scientific machine learning
AMS: 65-01, 65-04, 65L99, 65M99, 65N99


1  INTRODUCTION
==============================================================================
Scientific Machine Learning (SciML) [4] uses NNs in place of traditional
discretisation. Two classes of NN approaches:
- Variational/energy form: deep Ritz [16,20], Galerkin [39].
- Strong form (with AD): PINN [47] — handles general PDEs, image-free,
  no need for variational principle.
PINNs solve inverse problems with minimum code change [47,48,51,21,13];
extended to IDEs, FDEs [42], SDEs [57,55,41,56].

CONTRIBUTIONS:
- DeepXDE library implementing PINN algorithms.
- Designed as both education and research tool.
- Supports complex domains via constructive solid geometry (CSG).
- Time-dependent PDEs as easily as steady-state (treat IC as Dirichlet BC
  on the spatio-temporal domain).
- Callback functions for monitoring/modifying training (e.g. Fourier-spectrum
  monitoring).
- Compact code mirroring the mathematical formulation.
- New RAR method (Section 2.8).


2  ALGORITHM AND THEORY OF PINNs
==============================================================================

2.1 Deep neural networks
------------------------
Considered: feed-forward NN (FNN ≡ MLP) and residual NN (ResNet).
L-layer FNN, N_ℓ neurons in layer ℓ. With activation σ:
  N_0(x) = x;      hidden:  N_ℓ(x) = σ(W_ℓ N_{ℓ-1}(x) + b_ℓ);
  output: N_L(x) = W_L N_{L-1}(x) + b_L.
Common activations: sigmoid, tanh, ReLU.

2.2 Automatic differentiation
-----------------------------
Four ways to compute derivatives: hand-coded analytical, finite difference,
symbolic, automatic (AD). In DL, derivatives via backpropagation [49] —
a specialised AD.
AD = repeated chain rule via one forward + one backward pass. ALL partial
derivatives computed in O(1) passes regardless of input dimension; FD needs
d_in+1 forward passes. AD avoids truncation errors of FD and quadrature
errors of variational forms.
Higher-order derivatives via recursive AD (n times for n-th order). Better:
Taylor-mode AD [8,9].
AD differentiates the NN, NOT the data — so PINNs tolerate noisy data [42].

2.3 PINNs for solving PDEs
---------------------------
PDE (eq 1):
  f(x; ∂u/∂x_i; ∂²u/∂x_i ∂x_j; …; λ) = 0,  x ∈ Ω
with B(u, x) = 0 on ∂Ω (Dirichlet/Neumann/Robin/periodic). Time = special
component of x; IC = special Dirichlet on the spatio-temporal domain.

PROCEDURE 1 (PINN algorithm):
  Step 1: Construct NN û(x; θ) parameterised by θ.
  Step 2: Specify training sets T_f (residual points in Ω) and T_b (on ∂Ω).
  Step 3: Define loss as weighted L² norm of equation residual + BC residual.
  Step 4: Minimise L(θ; T) via gradient-based optimiser.

Loss (eq 2):
  L(θ; T) = w_f L_f(θ; T_f) + w_b L_b(θ; T_b)
  L_f = (1/|T_f|) Σ_{x∈T_f} ‖f(x; ∂û/∂x_i; …; λ)‖²
  L_b = (1/|T_b|) Σ_{x∈T_b} ‖B(û, x)‖²

Optimisers: gradient descent, Adam [29], L-BFGS [12].
EXPERIENCE (KEY for our paper):
- For SMOOTH PDE solutions, L-BFGS converges in fewer iterations than Adam
  (uses 2nd-order info).
- For STIFF solutions, L-BFGS often stuck in bad local minima.
Standard recipe: Adam first, then switch to L-BFGS. Adaptive activation
functions can help [24, 23].
PINNs have NO uniqueness guarantee — non-convex optimisation. Tune
hyperparameters; train from N (e.g. 10) random initialisations and pick
the lowest-loss model [42, 51, 21].

Three sampling strategies for residual points T:
  1. Fix T at the start (grid or random).
  2. Re-sample randomly each iteration (≡ mini-batch with infinite T).
  3. Adaptive refinement (Section 2.8 RAR).

Hard-constraint trick for simple BCs: use surrogate u_hat(x) = x(x-1)·N(x)
to enforce u(0)=u(1)=0 automatically [30].

NN learns from low to high frequency for function approximation [46, 54];
but PINNs (with high-order derivative terms in the loss) learn ALL
frequencies simultaneously — Fig 2 example: NN approximating
f(x)=Σ_{k=1}^{5} sin(2kx)/(2k) learns low frequencies first, but PINN
solving Poisson −f_xx = Σ 2k sin(2kx) learns all frequencies almost at
once, often faster than function approximation.

2.4 Approximation theory and error analysis
-------------------------------------------
THEOREM 2.1 (Pinkus [44]): Let m_i ∈ Z_+^d, m = max_i |m_i|, σ ∈ C^m(R)
non-polynomial. Then the space of single-hidden-layer NNs M(σ) is dense
in C^{m_1,…,m_s}(R^d) — for any f, compact K, ε>0, there exists g ∈ M(σ)
with max_{x∈K} |D^k f(x) − D^k g(x)| < ε for all k ≤ some m_i.
⇒ FNNs with enough neurons can simultaneously and uniformly approximate
any function AND its partial derivatives.

ERROR DECOMPOSITION (Bottou-Bousquet [11]):
  E = ‖ũ_T − u‖ ≤ ‖ũ_T − u_T‖_{E_opt} + ‖u_T − u_F‖_{E_gen} + ‖u_F − u‖_{E_app}
where:
  u_F = best approx in family F;
  u_T = global minimiser of L on training set T;
  ũ_T = optimiser's actual output.
- E_app: approximation error (function-class capacity).
- E_gen: generalisation error (number/locations of residual points + capacity).
- E_opt: optimisation error (optimiser, lr, iterations).
Larger NN ⇒ smaller E_app but higher E_gen (bias-variance tradeoff).
NO RIGOROUS PINN ERROR ESTIMATES YET; even quantifying these three errors
in supervised learning is open [36, 35, 25].

2.5 PINN vs FEM (Table 2)
--------------------------
                 PINN                        FEM
  Basis        | Neural net (nonlinear)    | Piecewise polynomial (linear)
  Parameters   | Weights & biases          | Point values
  Train pts    | Scattered (mesh-free)     | Mesh points
  PDE embed    | Loss function             | Algebraic system
  Solver       | Gradient-based optimiser  | Linear solver
  Errors       | E_app, E_gen, E_opt       | Approximation/quadrature
  Error bounds | Not yet                   | Partially available [14, 26]
PINN provides a nonlinear approximation to the function and its derivatives;
FEM provides a linear approximation.

2.6 PINNs for IDEs
-------------------
Use AD for integer-order derivatives; approximate integrals by classical
quadrature (Gaussian). Adds a fourth error: discretisation error E_dis [42].
Example: dy/dx + y(x) = ∫_0^x e^{t-x} y(t) dt
⇒ approximate the integral by Gauss-Legendre with degree n, then PINN solves
  dy/dx + y(x) ≈ Σ_i w_i e^{t_i(x) − x} y(t_i(x)).
Same idea works for FDEs and SDEs.

2.7 PINNs for inverse problems
-------------------------------
Unknown parameters λ in the PDE; have extra observations on T_i ⊂ Ω:
  I(u, x) = 0 on T_i.
Add an extra loss term:
  L(θ, λ; T) = w_f L_f + w_b L_b + w_i L_i,
  L_i(θ, λ; T_i) = (1/|T_i|) Σ ‖I(û, x)‖².
Optimise θ AND λ jointly: (θ*, λ*) = argmin L.

2.8 Residual-based Adaptive Refinement (RAR)
---------------------------------------------
Random T may be inefficient for solutions with steep gradients (Burgers).
RAR adds residual points where the residual is largest (analogous to FEM
adaptive refinement [2]).

PROCEDURE 2 (RAR):
  Step 1: Start with random T; train PINN for limited iterations.
  Step 2: Estimate mean PDE residual E_r (eq 3) by Monte Carlo on a
          random pool S = {x_1,…,x_|S|}:
            E_r ≈ (1/|S|) Σ ‖f(x; …)‖
  Step 3: STOP if E_r < E_0. Else add the m points with the LARGEST
          residuals from S into T; go to Step 2.

This is the original "RAR" — later renamed RAR-G (Greedy) by Wu et al. 2023.


3  DeepXDE USAGE AND CUSTOMISATION
==============================================================================

3.1 Usage
----------
PROCEDURE 3:
  Step 1: Specify domain (geometry module).
  Step 2: Specify PDE (TensorFlow grammar).
  Step 3: Specify BC and IC.
  Step 4: Combine via data.PDE (steady) or data.TimePDE (time-dep).
  Step 5: Construct NN via maps module.
  Step 6: Define Model = (problem, network).
  Step 7: Model.compile(optimiser, lr, loss_weights).
  Step 8: Model.train(restore_path=…); use callbacks for monitoring.
  Step 9: Model.predict(x).

Built-in primitive geometries: interval, triangle, rectangle, polygon,
disk, cuboid, sphere. Boolean ops on geometries (union |, difference -,
intersection &) ⇒ constructive solid geometry (CSG); 2-D and 3-D.

Built-in BCs: DirichletBC, NeumannBC, RobinBC, PeriodicBC, OperatorBC
(general). IC.
NNs: maps.FNN, maps.ResNet.
Loss types, metrics, optimisers, lr schedules, initialisers, regularisers
all configurable.

DeepXDE also approximates functions from multi-fidelity data [40] and
learns nonlinear operators (DeepONet) [34].

3.2 Customisability
-------------------
3.2.1 Geometry: subclass Geometry; implement inside / on_boundary /
      boundary_normal / periodic_point / random_points.
3.2.2 Neural networks: subclass Map; implement inputs / outputs / targets
      / build.
3.2.3 Callbacks: ModelCheckpoint, OperatorPredictor, FirstDerivative,
      MovieDumper. Subclass Callback and override on_epoch_begin /
      on_epoch_end / etc.


4  DEMONSTRATION EXAMPLES
==============================================================================
All examples: tanh activation, w_f = w_b = w_i = 1.
Hyperparameters (Table 3, depth × width × optimiser × lr × iterations):
  4.1 Poisson L-shape : 4 × 50 × Adam→L-BFGS × 0.001 × 50 000
  4.2 Burgers (RAR)   : 3 × 20 × Adam→L-BFGS × 0.001 × 15 000
  4.3 Lorenz inverse  : 3 × 40 × Adam        × 0.001 × 60 000
  4.4 Diff-react inv  : 3 × 20 × Adam        × 0.001 × 80 000
  4.5 Volterra IDE    : 4 × 20 × L-BFGS      ×   —   ×   —

4.1 Poisson on L-shaped domain
-------------------------------
  −Δu = 1 on Ω = [-1,1]² \ [0,1]²;  u = 0 on ∂Ω.
  1200 + 120 random points for T_f, T_b. Compared against spectral element
  method (SEM) [27]. Good visual agreement.

4.2 RAR for Burgers
--------------------
1-D Burgers:
  u_t + u u_x = ν u_xx,  ν = 0.01/π, x∈[-1,1], t∈[0,1]
  u(x,0) = -sin(πx), u(±1,t)=0.
Initial 2500 random T_f; RAR with m=1, E_0 = 0.005 ⇒ +40 points adaptively.
PINN+RAR captures the discontinuity much better than 2540 randomly sampled
points (Fig 8A). FD with 200×100 grid achieves a good solution (blue);
60×40 grid oscillates around the discontinuity. RAR-added points cluster
near the sharp interface.

2-D Burgers (coupled u, v) at Re = 5000:
  u_t + u u_x + v u_y = (1/Re)(u_xx + u_yy)
  v_t + u v_x + v v_y = (1/Re)(v_xx + v_yy)
Exact solution available [3]. 200 initial T_f + 10 RAR points (= 210 total)
beats 210 uniformly random T_f.

4.3 Lorenz inverse
-------------------
  dx/dt = ρ(y - x);  dy/dt = x(σ - z) - y;  dz/dt = xy - βz
True (ρ, σ, β) = (10, 15, 8/3); IC = (-8, 7, 27); observe at certain times
on [0, 3] (RK45). 400 random T_f + 25 equispaced T_i.
Recovered: (ρ, σ, β) = (10.002, 14.999, 2.668).

4.4 Diff-reaction inverse (porous media)
-----------------------------------------
  ∂C_A/∂t = D ∂²C_A/∂x² − k_f C_A C_B²
  ∂C_B/∂t = D ∂²C_B/∂x² − 2 k_f C_A C_B²
True D = 2e-3, k_f = 0.1. 40 000 observations. Recovered D = 1.98e-3,
k_f = 0.0971.

4.5 Volterra IDE
-----------------
  dy/dx + y(x) = ∫_0^x e^{t-x} y(t) dt;  y(0) = 1.
Exact: y(x) = e^{-x} cosh x. Gauss-Legendre quadrature degree 20.
12 equispaced residual points. Rel L² error 2e-3.


5  CONCLUDING REMARKS
==============================================================================
PINNs are mesh-free (AD), tolerate noisy data, easily extend to IDEs,
inverse problems. RAR improves training efficiency.
DeepXDE: compact PINN library mirroring the math.

LIMITATIONS:
- Slower than FEM for forward problems (mitigatable via offline training
  [58, 53] or time-parallel methods).
- Architecture search currently empirical; meta-learning [59, 17] could
  automate.
- Strong-form is easy via AD; weak/variational alternatives possible
  but require quadrature grids.
- Future: composite networks (CNN + PINN sharing parameters) for mixed
  data (images + scattered measurements) — geophysics, biomedicine.
